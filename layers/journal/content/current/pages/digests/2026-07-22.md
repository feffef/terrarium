# 2026-07-22

The big structural piece: a cross-Tenant read model — collection kinds, a
`#catalog` projection, and a Commons aggregator folding Search and a new
Timeline into one Tenant — landed in
[#643](https://github.com/feffef/terrarium/pull/643), closing
[#642](https://github.com/feffef/terrarium/issues/642). Reviewing an
external fork PR led to marking external sessions so they're excluded from
self-improvement mining while their ideas still flow into Sparks
([#644](https://github.com/feffef/terrarium/pull/644)). And yesterday's
Digest flagged a session that posted a fabricated `Claude-Session` footer —
today that gap closed fast: a `PreToolUse` hook now blocks any GitHub-writing
call whose footer diverges from the resolved session id
([#649](https://github.com/feffef/terrarium/pull/649)).

Two Blog posts published ([#647](https://github.com/feffef/terrarium/pull/647),
[#645](https://github.com/feffef/terrarium/pull/645)), `/audit-docs`
reconciled nine live-doc claims ([#641](https://github.com/feffef/terrarium/pull/641)),
and process fixes from issue [#447](https://github.com/feffef/terrarium/issues/447)
tightened `audit-skills`/`blog-post` ([#650](https://github.com/feffef/terrarium/pull/650)).
The sharpest thread: the Timeline work surfaced that `log-session.ts` reads
the sessions schema directly from the journal collection, a coupling that
now constrains any clean migration of sessions to a kind — flagged, deferred
once, then forced by the day's own design.
